RECOMMENDED ARCHITECTURE
================================================================================

Pattern: Deterministic Mock Data Factory with Seeded Random + AsyncStorage

Layer 1: Seeded Random Generator
  → LinearCongruentialGenerator with seed 20250101
  → Deterministic: same seed = same sequence always

Layer 2: Domain-Specific Factories
  → User Factory (creates mock-user-12345)
  → Domain Profile Factory (Chess, Valorant, Speedrunning)
  → Event Factory (8 events per domain)

Layer 3: Mock Data Service
  → Central singleton managing all mock data
  → Caches in AsyncStorage (survives app restart)
  → Auto-initializes when MOCK_AUTH=true

Layer 4: API Interceptor
  → Intercepts API calls at apiClient level
  → Returns mock response if MOCK_AUTH=true
  → Falls through to real API if MOCK_AUTH=false
  → Simulates realistic network latency

Layer 5: Environment Configuration
  → MOCK_AUTH env variable controls mode
  → Set in .env file
  → Passed through app.config.js

ALTERNATIVES CONSIDERED
================================================================================

1. Faker.js Library
   Score: 7/10
   Pros: Industry standard, extensive providers
   Cons: 50KB dependency, less control, overkill for MVP
   Decision: REJECTED - Use for 10+ domains later

2. Static JSON Files
   Score: 5/10
   Pros: Human-editable, consistent
   Cons: Poor scalability, file bundling issues
   Decision: REJECTED - Too rigid for gaming data

3. Backend Mock API Server
   Score: 8/10
   Pros: Production-like, error scenarios
   Cons: 2+ hours setup, infrastructure, not offline
   Decision: REJECTED - Overkill for MVP, consider for teams

4. Mock Service Worker (MSW)
   Score: 7/10
   Pros: Industry standard, network-level interception
   Cons: React Native limitations, learning curve
   Decision: REJECTED - Good for later, simpler solution works now

5. Seeded Random Generation (CHOSEN)
   Score: 9.5/10
   Pros: Simple, zero-dep, deterministic, offline, testable
   Cons: Requires initial setup code
   Decision: SELECTED - Perfect for MVP, scalable, maintainable
